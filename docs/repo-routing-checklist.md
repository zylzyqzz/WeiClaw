# Repo Routing Checklist

- [ ] Is this feature part of the public mainline WeiClaw product?
- [ ] Does this capability involve ownership recognition, long-term memory enhancement, device management backends, or commercial logic?
- [ ] Would publishing this code directly expose private core value?
- [ ] Does this change preserve the single unified install entry through WeiClaw?
- [ ] Has the target repo been declared before work starts?
- [ ] Has the push target repo been written explicitly before running `git push`?
- [ ] Are any private paths, tokens, keys, or production configs being introduced into the public repo?
